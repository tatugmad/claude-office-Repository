# Claudeコード環境別ルール

> **このファイルはClaudeコード専用です。Claudeチャット環境では読み込み禁止。**
> CLAUDE.mdの環境判定で `CLAUDE_ENV=chat` が検出されなかった場合にのみ読み込むこと。

## ファイル操作

- gitコマンドを使用してファイルの読み書き・commit & pushを行う
- commit & push の前に必ず `git pull` を実行し、最新状態を取得すること
- commit メッセージの本文に session URL を含める（取得可能な場合）:
  ```
  - session: https://claude.ai/code/session_XXXXX
  ```

## 一時ファイル

- Pythonスクリプト等の一時ファイルは作業フォルダ内の `_temp/` に作成する
- `_temp/` は `.gitignore` で除外し、リポジトリには commit しない

## VM環境

### 特性

- クラウドVMはセッションごとに使い捨て。予告なく消滅しうる
- commit & push していないファイルはVM消滅とともに失われる。常にpush済み状態を維持すること
- ツールのインストール状態もセッションをまたいで保持されない

### ツールの自動導入

`.devcontainer/devcontainer.json` が存在する場合、VM起動時にそれに基づき必要なツールが自動インストールされる。

セッション中に新たなツールが必要になった場合:
1. VM上に手動でインストールして作業を続行する
2. 今後も継続的に必要であれば、devcontainer.json への追加（未作成の場合は新規作成）をユーザーに提案する

## UIツール

- 作業フォルダ選択などの選択UIには **AskUserQuestionツール** を使い、インタラクティブな選択UIで提示すること（テキストの番号リストで代替しない）
- Agentツール呼び出し時は `description` を日本語で記述し、プロンプトにも「日本語で回答せよ」を含めること（UI表示の日本語化）

## DOCX編集の注意

- DOCX（OOXML）のXML編集にPython ElementTreeを使ってはならない。ElementTreeはXMLを書き直す際に名前空間宣言を変更し、Word XMLが無効になる。必ず文字列置換（`str.replace` / `re.sub`）で行うこと

## タスクキュー（実行）

セッション開始時に `_claude/tasks/pending/` に未実行の指示ファイルがある場合:

1. 番号が最も小さいものを1件読み込む
2. 内容をユーザーに要約し、「この指示を実行してよいですか？」と確認する
3. **ユーザーの承認なしに実行しない**
4. 承認されたら指示に従って実行する
5. 指示ファイルの「実行結果」セクションに、実行日時・結果・成果物・問題や備考を追記する
6. 指示ファイルを `review/` に移動し、commit & push する
7. ユーザーに「Claudeチャットに結果を確認してもらってください」と伝える
8. 次のファイルがあれば「次の指示があります。続けますか？」と確認する
9. ユーザーが「後で」「今日はやめておく」等と回答した場合は、通常のセッション開始フロー（作業フォルダ選択）に進む

## 外部リポジトリでの作業

指示ファイルに「対象リポジトリ」が記載されている場合、本リポジトリ（OfficeWork）ではなく指定された外部リポジトリで作業する。

### 操作手順

1. 外部リポジトリをVM上にcloneする（本リポジトリとは別ディレクトリ）
2. 指示ファイルの「ブランチ運用」に従う:
   - **main直接commit**: 本リポジトリと同じくmainに直接commit & push
   - **ブランチ作成→PR**: 指示に従いブランチを作成し、作業完了後にPRを作成する。ブランチ命名規則・PR作成方法は指示ファイルの「備考」に従う
3. 外部リポジトリでの作業完了後、本リポジトリに戻り、指示ファイルの「実行結果」に成果（ブランチ名、commitハッシュ、PR URL等）を記録する
4. 指示ファイルを `review/` に移動し、commit & push する

### 注意事項

- 外部リポジトリにCLAUDE.mdがある場合は必ず読み、そのリポジトリ固有のルール（設計原則、コーディング規約等）に従うこと
- 本リポジトリ（OfficeWork）のCLAUDE.mdと外部リポジトリのCLAUDE.mdが矛盾する場合は、技術的なルール（コーディング規約等）は外部リポジトリを優先し、運用ルール（タスクキュー、commit報告等）は本リポジトリを優先する

## project.md の自動取得

プレースホルダが残っている場合、`git remote -v` を実行しリモートURLからユーザー名とリポジトリ名を抽出する。URL形式: `https://github.com/{ユーザー名}/{リポジトリ名}.git`
