# Claudeチャット環境別ルール

> **このファイルはClaudeチャット専用です。Claudeコード環境では読み込み禁止。**
> プロジェクト指示に `CLAUDE_ENV=chat` が含まれている場合にのみ読み込むこと。

## ファイル操作

- GitHub API を使用してファイルの読み書き・commitを行う
- プロジェクト指示（システムプロンプト）内の Personal Access Token を認証に使用する
- API経由のcommitでは `git pull` は不要（APIは常に最新のリポジトリ状態に対して操作する）

## UIツール

- 作業フォルダ選択などの選択UIには、チャット環境で利用可能な選択肢提示機能を使うこと

## タスクキュー（作成・レビュー）

### セッション開始時の確認（通常）

- `_claude/tasks/pending/` に未実行の指示ファイルがある場合: 「未実行の指示が○件あります。Claudeコードで実行してください」と通知する
- `_claude/tasks/review/` にレビュー待ちの指示ファイルがある場合: 内容を読み、実行結果をレビューする。問題なければ対象作業フォルダの `done/YYYY-MM/` に移動し、commit & push する。問題があれば修正した指示ファイルを `pending/` に再配置する

### 作業フォルダ選択後の確認（ソースコード用リポジトリがある場合）

`_context.md` に「ソースコード用リポジトリ」が記載されている場合、GitHub API でそのリポジトリの `_tasks/` を確認する:

- `_tasks/pending/` に未実行の指示ファイルがある場合: 「未実行の指示が○件あります。Claudeコードで実行してください」と通知する
- `_tasks/review/` にレビュー待ちの指示ファイルがある場合: 内容を読み、実行結果をレビューする。問題なければ本リポジトリの作業フォルダの `done/YYYY-MM/` に移動し、ソースコード用リポの `_tasks/review/` からは削除する。問題があれば修正した指示ファイルをソースコード用リポの `_tasks/pending/` に再配置する

### 指示ファイルの作成

ユーザーの要望がClaudeコードで実行すべき作業と判断した場合:

1. 指示ファイルをCLAUDE.mdの形式に従って作成する
2. 配置先を判断する:
   - ソースコード用リポジトリがある作業 → そのリポジトリの `_tasks/pending/` に配置（GitHub API使用）
   - 通常の作業 → `_claude/tasks/pending/` に配置
3. commit & push する
4. ユーザーに「Claudeコードで実行してください」と伝える

**チャットではタスクの実行を行わない**。タスクの実行はClaudeコードで行う。

## project.md の自動取得

プレースホルダが残っている場合、プロジェクト指示（システムプロンプト）内の「対象リポジトリ:」行からユーザー名とリポジトリ名を抽出する。形式: `- {ユーザー名}/{リポジトリ名}（Private、本番）`
